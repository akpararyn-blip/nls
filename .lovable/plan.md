# Блок «Превосходная техподдержка 24/7» на серверных страницах

Перед `<RelatedServices ... />` на пяти страницах:
- `src/routes/dedicated.tsx`
- `src/routes/vps.tsx`
- `src/routes/colocation.tsx`

Один и тот же компонент, без пропсов — контент идентичный.

## Что внутри блока

Двухколоночная секция (на мобиле — одна колонка, тикеты под текстом):

**Левая колонка — текст и метрики**
- Eyebrow: «Превосходная техподдержка»
- Крупный заголовок: «24/7»
- Подзаголовок-описание (2–3 строки RU/KZ через `useT()`):
  «Наши инженеры бесплатно решают любые технические вопросы… доступны даже в выходные и праздники.»
- Сетка из 3 метрик (карточки со стеклянным эффектом, в одном стиле с уже существующими `trust-card` / `related-card`):
  - **25+** — человек в команде поддержки
  - **3:45 мин** — среднее время реакции
  - **4.55** — средняя оценка качества

**Правая колонка — анимация тикетов**

Контейнер фиксированной высоты (~440px десктоп / ~360px мобайл) с `overflow: hidden`, мягким градиентным fade сверху и снизу. Внутри — бесконечная вертикальная marquee-лента карточек-тикетов, движение строго снизу вверх.

Карточка тикета:
- Тёмная/светлая подложка под общий стиль сайта (через CSS-переменные темы), скругление, тонкая рамка, мягкая тень.
- Шапка: `Ticket #DDMMYYYY-HHMMSS` (моноширинный шрифт) + бейдж статуса справа.
- Тело: 2 строки «описания проблемы», заблюренные через `filter: blur(4px)` + полупрозрачная серая плашка-плейсхолдер (имитация скрытых персональных данных).
- Футер: имя клиента в виде инициалов в кружке + заблюренная строка «email/phone».
- Статус по умолчанию — **«В работе»**, желтый индикатор + точка.
- Когда центр карточки пересекает середину контейнера — статус анимированно меняется на **«Решено»**, бейдж и рамка плавно подсвечиваются зеленым, появляется иконка-галочка. Используем `IntersectionObserver` или отслеживание `getBoundingClientRect()` в `requestAnimationFrame` относительно центра контейнера; флаг `resolved` хранится в локальном состоянии по id тикета.

Поведение анимации:
- Бесконечный marquee: дублируем массив тикетов дважды, анимируем `translateY` от `0` до `-50%` через CSS `@keyframes` (`animation: ticketsScroll 28s linear infinite`).
- Hover на контейнере не останавливает (поток real-time), но `prefers-reduced-motion` — отключает анимацию (статика, все тикеты «Решено»).
- ID генерируются один раз при монтировании: берём `new Date()`, для каждого тикета вычитаем нарастающее количество секунд (например, 7, 19, 34…), форматируем `DDMMYYYY-HHMMSS`. Получаем реалистичную последовательность таймстампов, выглядящую как real-time.

## Реализация

### Новый компонент `src/components/nls/SupportPromo.tsx`
- Самодостаточный, без внешних библиотек анимации (CSS keyframes + минимальный JS для смены статуса).
- Использует `useT()` для RU/KZ.
- Внутри — подкомпонент `TicketCard` с props `{ id, name, problem, resolved }`.
- Хук, который через `requestAnimationFrame` раз в ~150 мс проверяет позиции карточек и помечает «пересекшие середину» как `resolved`. После завершения цикла marquee сбрасываем флаги для следующего прохода.

### Мок-данные тикетов
- 6 тикетов с реалистичными RU-фразами проблем (текст всё равно заблюрен, но для семантики). Имена — обычные RU-инициалы (А. Петров, М. Сулейменов и т.п.).
- Дублируются для бесшовного цикла.

### CSS — добавляем в `src/styles/nls.css`
Новые классы:
- `.support-promo`, `.support-promo__inner` (grid 1.1fr / 1fr, gap),
- `.support-promo__eyebrow`, `.support-promo__title` (очень крупный «24/7»), `.support-promo__desc`,
- `.support-metrics` (3 колонки → 1 на мобиле), `.support-metric`, `.support-metric__value`, `.support-metric__label`,
- `.support-tickets` (контейнер с fade-маской через `mask-image: linear-gradient(...)`),
- `.support-tickets__track` (анимация `ticketsScroll`),
- `.ticket-card`, `.ticket-card--resolved` (плавный transition border-color + box-shadow на акцентный зелёный),
- `.ticket-card__head`, `.ticket-card__id` (monospace), `.ticket-card__status` + модификаторы `--inwork` / `--resolved`,
- `.ticket-card__body` с `.ticket-card__line` (заблюренная плашка),
- `.ticket-card__foot` с инициалами и blur-строкой,
- `@keyframes ticketsScroll { from { transform: translateY(0) } to { transform: translateY(-50%) } }`,
- Медиа `@media (max-width: 768px)`: одна колонка, метрики в строку с горизонтальным скроллом или 3 в ряд (на выбор по месту), высота тикетов уменьшается,
- `@media (prefers-reduced-motion: reduce)`: `animation: none`, все тикеты в состоянии resolved.

### Интеграция в страницы
В каждом из 5 файлов:
1. Добавить `import { SupportPromo } from "@/components/nls/SupportPromo";`
2. Вставить `<SupportPromo />` прямо перед `<RelatedServices ... />`.

## Технические детали
- Без новых зависимостей. Lucide уже в проекте — иконки `Check`, `Clock`, `Users`, `Star`, `Headphones`.
- Доступность: `role="region"` + `aria-label`, для marquee — `aria-hidden="true"` на дубликате треков, статусы внутри `aria-live="off"` (поток декоративный).
- SSR-safe: генерация ID и `Date.now()` — внутри `useMemo` с пустыми зависимостями, никакого расхождения SSR/CSR (компонент чисто клиентский в Vite SPA, но всё равно безопаснее).
